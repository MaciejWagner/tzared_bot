# F5.T1: Template Capture Tool

## Agent: QA_INTEGRATION

## Task Overview
Create a tool to capture and save template images from the game for use in template matching.

## Context
Create tool in `tools/TemplateCapturer/`.

## Requirements

1. Create console application with commands:
   - `--capture <name> --region <x,y,w,h>` - Capture region
   - `--capture <name> --fullscreen` - Capture full screen
   - `--list` - List templates
   - `--verify` - Verify all templates exist

2. Implement capture functionality:
   - Wait for user to position game
   - Capture using DxgiScreenCapture
   - Extract region if specified
   - Save as PNG

3. Create template manifest (JSON):
   ```json
   {
     "templates": {
       "victory": { "filePath": "...", "region": null, "matchThreshold": 0.85 },
       "defeat": { "filePath": "...", "matchThreshold": 0.85 },
       "menu_main": { "region": {"x":0,"y":0,"width":400,"height":100} },
       "in_game_minimap": { "matchThreshold": 0.7 },
       "loading": { "matchThreshold": 0.75 }
     }
   }
   ```

4. Create verification command:
   - Check all template files exist
   - Validate image dimensions
   - Report missing/corrupted files

## Outputs
- `tools/TemplateCapturer/TemplateCapturer.csproj`
- `tools/TemplateCapturer/Program.cs`
- `assets/templates/template_manifest.json`
- (After manual capture): `assets/templates/*.png`

## Usage
```bash
dotnet run --project tools/TemplateCapturer -- --capture victory --fullscreen
dotnet run --project tools/TemplateCapturer -- --verify
```

## Manual Steps After Implementation
1. Run the game
2. Navigate to each screen state (victory, defeat, menu, in-game)
3. Use tool to capture each template
4. Verify all templates are saved
